CentOS 安装

原本安装的是最小化版本无可视化界面，但是发现 VMWare 里最小化版本的复制粘贴简直是个坑，最后返回了图形化界面的怀抱。

  1. 安装 VMWare 和 CentOS 步骤和其他在 VMWare 上安装系统步骤一致

  2. 打开虚拟机，点击 Install Centos7

  3. 语言选择中文

  4. 选择软件安装

    选择GNOM桌面、开发者工具

  5. 选择安装位置

    其他存储选项，我要配置分区

    手动分区，选择标准分区，添加新挂载点

    挂载点：/boot 300MB、swap 2GB、/ 10GB

    创建完挂载点后点击完成；并接受更改

  6. 关闭kdump

  7. 配置网络与主机名

    查看自己电脑的 ipconfig，查看（以太网适配器）VMnet1 和 VMnet8

    手动配置地址，本机 ipv4 和 地址、网关同一个子网即可

  8. 点击安装

  9. 设置用户名、密钥

Docker

Docker基于镜像，可以秒启各种容器，每种容器都是完整、互相隔离的运行环境。

直接按照 Docker 文档 进行操作：下述操作均为 2023 年 6 月版本，以文档最新版为准del>还是根据菜鸟教程走吧

  1. 卸载旧版本

    sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine

    当然，因为我们之前没有安装，所以会返回 No Matching

  2. 安装依赖包

    sudo yum install -y yum-utils \
  device-mapper-persistent-data \
  lvm2

  3. 配置镜像仓库

    这里不要用官方源，后面会出大问题

    yum-config-manager \
    --add-repo \
    http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

  4. 安装 Docker

    这里用和教程一样的版本，最新的版本似乎和 CentOS 不兼容

    sudo yum install docker-ce docker-ce-19.03.2-3.e17

    一直 y

  5. 启动 Docker

    sudo systemctl start docker

    可以使用 docker -v 检查版本

  6. 开机自启动

    目前 Docker 每次开机需要重新运行上述命令启动，因此设置开机自启动

    sudo systemctl enable docker

镜像加速器

Docker 大多从国外 Docker Hub 下载容器，但是这样速度比较慢，所以可以使用阿里云的镜像服务

  1. 选择容器镜像服务的镜像加速器

  2. 选择 CentOS 的操作文档

  3. 创建镜像目录

sudo mkdir -p /etc/docker
  1. 配置镜像加速器地址
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://j5a675zg.mirror.aliyuncs.com"]
}
EOF
  1. 重启 Docker 后台线程
sudo systemctl daemon-reload
  1. 重启 Docker 服务
sudo systemctl restart docker

Docker 安装 MySQL

  1. 在 Docter Hub 中搜索 MySQL，可以在 TAG 中看见所有可操作版本

  2. 下载MySQL 5.7

    sudo docker pull mysql:5.7

  3. 使用 sudo docker images 检查已安装镜像

  4. 启动 MySQL

    sudo docker run -p 3306:3306 --name mysql \
-v /mydata/mysql/log:/var/log/mysql \
-v /mydata/mysql/data:/var/lib/mysql \
-v /mydata/mysql/conf:/etc/mysql \
-e MYSQL_ROOT_PASSWORD=root \
-d mysql:5.7

    参数说明：

    • -name：指定容器名称
    • -v：目录挂载，将主机的文件夹和容器中的文件夹映射
    • -p：指定端口映射，即将主机端口和容器端口进行映射
    • -e：设置 MySQL 参数
    • -d：后台运行

  5. 查看运行的容器

    docker ps

  6. 修改配置 vi /mydata/mysql/conf/my.conf

    添加内容

    [client]
default-character-set=utf8
[mysql]
default-character-set=utf8
[mysqld]
init_connect='SET collation_connection = utf8_unicode_ci'
init_connect='SET NAMES utf8'
character-set-server=utf8
collation-server=utf8_unicode_ci
skip-character-set-client-handshake
skip-name-resolve

  7. 重启 MySQL

    docker restart mysql

Docker 安装 Redis

  1. 下载 Redis

    docker pull redis

  2. 如果直接挂载的话docker会以为挂载的是一个目录，所以我们先创建一个文件然后再挂载

    mkdir -p /mydata/redis/conf
touch /mydata/redis/conf/redis.conf

  3. 运行

    docker run -p 6379:6379 --name redis \
-v /mydata/redis/data:/data \
-v /mydata/redis/conf/redis.conf:/etc/redis/redis.conf \
-d redis redis-server /etc/redis/redis.conf

  4. 设置持久化

    vim /mydata/redis/conf/redis.conf

    编写以下内容

    appendonly yes

    重新启动

    docker restart redis

  5. 设置自启动

    docker update redis --restart=always

Docker 安装 mvn

  1. 移动到 /usr/app/ 

    cd /usr/app/

  2. 新建 maven 文件夹

    mkdir maven/

  3. 移动到 maven/

    cd maven/

  4. 下载 maven 到系统

    wget https://archive.apache.org/dist/maven/maven-3/3.6.1/binaries/apache-maven-3.6.1-bin.tar.gz

  5. 解压 maven

    tar -zxvf apache-maven-3.6.1-bin.tar.gz

    解压完成后会看到 apache-maven-3.6.1 文件目录。

  6. 配置环境变量，修改 /etc/profile

    vim /etc/profile

  7. 在文件尾添加信息

    export MAVEN_HOME=/usr/app/maven/apache-maven-3.6.1
export PATH=$PATH:$MAVEN_HOME/bin

  8. 重启环境变量，使之生效

    source /etc/profile

  9. 验证

    mvn -v

  10. 打开 settings 文件

    vim /usr/app/maven/apache-maven-3.6.1/conf/setting.xml

  11. 配置阿里云镜像，配置 jdk1.8

    <mirrors>
  <mirror>
    <id>nexus-aliyun</id>
    <mirrorOf>central</mirrorOf>
    <name>Nexus aliyun</name>
    <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
  </mirror>
</mirrors>


<profile>
  <id>jdk-1.8</id>

  <activation>
    <activeByDefault>true</activeByDefault>
    <jdk>1.8</jdk>
  </activation>

  <properties>
    <maven.compiler.source>1.8</maven.compiler.source>
    <maven.compiler.target>1.8</maven.compiler.target>
    <maven.compiler.compilerVersion>1.8</maven.compiler.compilerVersion>
  </properties>
</profile>
